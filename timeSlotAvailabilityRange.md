# `GET /time-slots/available-range`

Возвращает доступные тайм-слоты за **диапазон дат**, сгруппированные по датам.

Отличия от `GET /time-slots/available`:

1. Принимает диапазон дат (`timeFrom` / `timeTo`) вместо одной `date`.
2. `companyId` определяется по bearer-токену и в запросе не передаётся.
3. Ответ группируется по датам.

---

## 1. HTTP-контракт

| Метод | Путь                          | Авторизация  |
|-------|-------------------------------|--------------|
| GET   | `/time-slots/available-range` | Bearer-токен |

### Параметры запроса (query)

| Имя         | Тип    | Обязательный | Примечания                                                                 |
|-------------|--------|--------------|----------------------------------------------------------------------------|
| `timeFrom`  | дата   | да           | Формат `Y-m-d`. Включительная нижняя граница диапазона.                    |
| `timeTo`    | дата   | да           | Формат `Y-m-d`. Включительная верхняя граница. Должно быть `>= timeFrom`.  |
| `longitude` | строка | нет          | Вместе с `latitude` используется для автоматического определения сектора.  |
| `latitude`  | строка | нет          | Вместе с `longitude` используется для автоматического определения сектора. |
| `typeId`    | int    | да           | Тип тайм-слота (например, доставка = 2).                                   |

### Заголовки

| Заголовок       | Обязательный | Назначение                                                    |
|-----------------|--------------|---------------------------------------------------------------|
| `Authorization` | да           | `Bearer <token>`. Из токена определяется `companyId` клиента. |
| `X-User`        | да           | Идентификатор пользователя (целое число).                     |

`companyId` **нельзя** передавать в query — он определяется по токену.

---

## 2. Ответ — `200 OK`

```json
{
  "data": [
    {
      "date": "2026-04-23",
      "timeSlots": [
        {
          "id": 153849,
          "timeFrom": "09:00:00",
          "timeTo": "10:00:00",
          "status": 51,
          "typeId": 2
        },
        {
          "id": 153850,
          "timeFrom": "10:00:00",
          "timeTo": "11:00:00",
          "status": 51,
          "typeId": 2
        }
      ]
    },
    {
      "date": "2026-04-24",
      "timeSlots": []
    }
  ]
}
```

### Поля элемента `data[]`

| Поле        | Тип                | Описание                                                |
|-------------|--------------------|---------------------------------------------------------|
| `date`      | строка (`Y-m-d`)   | Дата.                                                   |
| `timeSlots` | массив тайм-слотов | Список доступных слотов на эту дату. Может быть пустым. |

### Поля тайм-слота

| Поле       | Тип              | Описание                                                                                                             |
|------------|------------------|----------------------------------------------------------------------------------------------------------------------|
| `id`       | int              | Идентификатор тайм-слота.                                                                                            |
| `timeFrom` | строка (`H:i:s`) | Время начала слота.                                                                                                  |
| `timeTo`   | строка (`H:i:s`) | Время окончания слота.                                                                                               |
| `status`   | int              | ID статуса тайм-слота. (51 - активный, 52 - не активный) *отправяем активные по дефолту если нету фильтра по статусу |
| `typeId`   | int              | ID типа тайм-слота. (2 - доставка)                                                                                   |

### Правила формирования ответа

- Каждая дата из диапазона `[timeFrom, timeTo]` встречается в ответе ровно один
  раз, по возрастанию, даже если доступных слотов нет (`timeSlots: []`).
- Для прошедших и сегодняшней даты `timeSlots` всегда пуст.
- Если по переданным координатам не определён сервисный сектор — для всех дат
  вернётся пустой `timeSlots`.
- Если функциональность тайм-слотов для компании отключена — для всех дат
  вернётся пустой `timeSlots`.

---

## 3. Ошибки

| Статус | Когда                                                                                                  |
|--------|--------------------------------------------------------------------------------------------------------|
| 401    | Отсутствует или невалидный bearer-токен.                                                               |
| 403    | У пользователя нет прав на просмотр доступных тайм-слотов, либо токен не привязан к компании.          |
| 422    | Ошибка валидации (отсутствует `timeFrom` / `timeTo` / `typeId`, неверный формат, `timeTo < timeFrom`). |

Формат тела ошибки соответствует стандартному ответу API.

Authorization: Bearer <token>
X-User: 42
